WITHOUT_SMOTE VERSION - ABOUT THIS EXPERIMENT

Owner:
- This folder contains the baseline cohort-specific variant without SMOTE.

Problem being solved:
- Binary target `target`:
  - `1` = cardiovascular disease present
  - `0` = cardiovascular disease absent
- The workflow trains separate models for age cohorts:
  - `Young`: age < 45
  - `Middle`: 45 <= age <= 65
  - `Elderly`: age > 65
- This version keeps original class distribution and is the control baseline.

Main scripts and what each does:

1) run_without_smote.py
- Entry-point for this version.
- Calls shared pipeline with `use_smote=False`.
- Trains from the generated processed training dataset in the parent project:
  - `..\datasets\processed\processed_train_100k_stratified.csv`
- Evaluates on the processed holdout test split:
  - `..\datasets\processed\processed_test.csv`
- Uses the raw dataset only as a reference for age cutoffs in processed space:
  - `..\datasets\raw\heart_statlog_cleveland_hungary_final(1).csv`
- Saves this version's output to:
  - `results/without_smote/results_without_smote.json`

2) ../shared_utils.py (shared by both versions)
- Core pipeline implementation.
- `load_and_clean_data(...)`
  - Loads CSV, drops duplicates, normalizes columns, removes invalid `st_slope == 0`.
- `assign_age_groups(...)`
  - Creates `Young`, `Middle`, `Elderly` cohorts using raw age.
- `run_variant(...)`
  - Raw-data mode kept for the original comparison flow.
- `run_variant_on_generated_data(...)`
  - Full generated-data flow:
    - load artificial processed train dataset
    - load processed holdout test dataset
    - recover age cutoffs in processed space from the raw reference dataset
    - split both datasets into `Young`, `Middle`, and `Elderly`
    - select top 10 training-only features per cohort
    - no SMOTE resampling (baseline)
    - per-cohort `GridSearchCV` tuning for XGBoost
    - per-cohort predictions and metrics
    - aggregate overall metrics
    - save JSON output
- `tune_xgboost_with_gridsearch(...)`
  - Per-cohort dynamic hyperparameter search with stratified CV.
  - Tuned parameters:
    - `n_estimators`
    - `max_depth`
    - `learning_rate`
    - `subsample`
  - CV scoring metric: `f1`
- `smote_resample_binary(...)`
  - Implemented in shared module but not used in this baseline variant.
- `compute_metrics(...)`
  - Computes Accuracy, Precision, Recall, F1, ROC-AUC, PR-AUC, TN/FP/FN/TP.

3) ../compare_smote_vs_no_smote.py
- Runs both versions with the same cohort-based pipeline.
- Produces:
  - overall metric comparison
  - per-cohort metric comparison
  - per-cohort best hyperparameters
  - markdown report `reports/SMOTE_COMPARISON_REPORT.md`
  - visualization PNGs in `visualizations/`
    - overall metrics
    - per-cohort metrics
    - confusion matrices
    - sampling comparison
    - selected-feature importance comparison

Techniques used in this non-SMOTE cohort-tuned baseline:

1) Leakage-safe preprocessing
- The generated train dataset and processed test dataset come from the parent project's leakage-safe preprocessing pipeline.
- Test rows remain outside the artificial training data generation step.

2) Cohort-specific modeling
- Independent model per age cohort.
- Allows each cohort to have different model complexity and learning dynamics.

3) Dynamic hyperparameter optimization (GridSearchCV)
- Removes fixed one-size-fits-all XGBoost settings.
- Finds best params per cohort using cross-validation.

4) No synthetic oversampling
- Class distribution is preserved in training data.
- Provides a clean reference to evaluate the true impact of SMOTE.

5) Cohort-wise feature selection
- Keeps the top 10 training-only features per cohort using XGBoost feature importance.
- Happens before the SMOTE / non-SMOTE split so the comparison stays fair.

6) Multi-level evaluation
- Reports both cohort-level and overall metrics.
- Includes confusion-matrix components for error analysis.

Important outputs for this version:
- `results/without_smote/results_without_smote.json`:
  - per-cohort class distribution and tuning metadata
  - per-cohort and overall metrics
- Used by `reports/SMOTE_COMPARISON_REPORT.md` during comparison.
