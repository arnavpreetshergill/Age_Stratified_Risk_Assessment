WITH_SMOTE VERSION - ABOUT THIS EXPERIMENT

Owner:
- This folder contains the SMOTE-enabled cohort-specific variant of the cardiovascular disease prediction pipeline.

Problem being solved:
- Binary target `target`:
  - `1` = cardiovascular disease present
  - `0` = cardiovascular disease absent
- The workflow now trains separate models for age cohorts because feature patterns can differ by age:
  - `Young`: age < 45
  - `Middle`: 45 <= age <= 65
  - `Elderly`: age > 65

Main scripts and what each does:

1) run_with_smote.py
- Entry-point for this version.
- Calls shared pipeline with `use_smote=True`.
- Saves this version's output to:
  - `with_smote/results_with_smote.json`

2) ../shared_utils.py (shared by both versions)
- Core pipeline implementation.
- `load_and_clean_data(...)`
  - Loads CSV, drops duplicates, normalizes column names, removes invalid `st_slope == 0` rows.
- `assign_age_groups(...)`
  - Assigns each row to `Young`, `Middle`, or `Elderly` using raw age.
- `run_variant(...)`
  - Full end-to-end flow:
    - leakage-safe train/test split
    - cohort split by age
    - per-cohort preprocessing (fit on train cohort only)
    - SMOTE on train cohort only (this version)
    - per-cohort `GridSearchCV` tuning for XGBoost
    - per-cohort prediction and metrics
    - aggregate overall metrics across cohorts
    - save JSON results
- `smote_resample_binary(...)`
  - Custom SMOTE logic:
    - Detect minority class in a cohort train split
    - Build minority-neighbor graph
    - Interpolate synthetic samples
    - Balance minority count up to majority count
- `tune_xgboost_with_gridsearch(...)`
  - Uses `GridSearchCV` with stratified CV for each cohort independently.
  - Tuned parameters:
    - `n_estimators`
    - `max_depth`
    - `learning_rate`
    - `subsample`
  - CV scoring metric: `f1`
  - This is dynamic and cohort-specific (not fixed globally).
- `compute_metrics(...)`
  - Computes Accuracy, Precision, Recall, F1, ROC-AUC, PR-AUC, TN/FP/FN/TP.

3) ../compare_smote_vs_no_smote.py
- Runs both variants (with and without SMOTE) using the same split and cohort logic.
- Produces:
  - overall metric comparison
  - per-cohort metric comparison
  - best hyperparameter table per cohort
  - markdown report `SMOTE_COMPARISON_REPORT.md`

Techniques used in this SMOTE cohort-tuned version:

1) Leakage-safe preprocessing
- Train/test split occurs before fitting preprocessors.
- Each cohort has its own preprocessor fitted only on that cohort's training rows.

2) Cohort-specific modeling
- Separate models for `Young`, `Middle`, and `Elderly`.
- Avoids forcing one parameter set to fit all age populations.

3) Class-imbalance handling with SMOTE
- SMOTE is applied only on training rows inside each cohort.
- Test set remains untouched.
- Preserves fair evaluation while balancing minority representation.

4) Hyperparameter optimization with GridSearchCV
- Per-cohort dynamic tuning finds best tree settings for that cohort.
- Better than using one fixed XGBoost configuration for all age groups.

5) Multi-level evaluation
- Metrics reported at two levels:
  - Cohort-level
  - Overall combined
- Includes threshold-dependent metrics and ranking metrics (ROC-AUC, PR-AUC).

Important outputs for this version:
- `results_with_smote.json`:
  - per-cohort sampling stats
  - per-cohort best GridSearch parameters
  - per-cohort and overall metrics
- Used by `SMOTE_COMPARISON_REPORT.md` when comparison script runs.
